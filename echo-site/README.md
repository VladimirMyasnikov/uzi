# Echo-site — витрина УЗИ

Next.js 16 приложение: каталог УЗИ-оборудования, услуги ремонта, контакты, лид-формы. Данные из локальной папки `catalog-data`.

---

## Как запустить

### Вариант 1: двойной клик (Windows)

Запустите **`start-dev.bat`** в папке `echo-site`. Скрипт установит зависимости и запустит сервер.

### Вариант 2: из терминала

```bash
cd echo-site
npm install
npm run dev
```

Откройте в браузере: **http://localhost:3000**

---

## Требования

- **Node.js 18+**. Проверка: `node -v`
- Скачать: https://nodejs.org/ (рекомендуется LTS)

---

## Скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Режим разработки (Turbopack), http://localhost:3000 |
| `npm run build` | Продакшен-сборка |
| `npm run start` | Запуск после `build` |
| `npm run lint` | ESLint |

## Структура

```
echo-site/
├── src/
│   ├── app/              # Страницы (App Router)
│   │   ├── page.tsx      # Главная
│   │   ├── catalog/       # Каталог: категории и товары
│   │   ├── services/     # Ремонт сканеров, датчиков
│   │   ├── contacts/     # Контакты
│   │   ├── faq/          # FAQ
│   │   ├── news/         # Новости
│   │   ├── specials/      # Спецпредложения
│   │   └── images/       # API раздачи картинок каталога
│   ├── components/       # UI: header, footer, галерея, тема
│   ├── config/           # site, data, themes
│   └── lib/              # catalog-loader (fs + JSON)
├── catalog-data/         # Категории/товары: папки + data.json + картинки
├── public/               # Статика (favicon и т.д.)
├── start-dev.bat         # Запуск для Windows
└── package.json
```

Каталог: каждая категория — папка, внутри — папки товаров с `data.json` и файлами изображений (jpg, png, webp). Изображения отдаются по маршруту `/images/{category}/{product}/{file}`.

## Если не запускается

1. **«node не является внутренней командой»** — установите Node.js и перезапустите терминал.
2. **Ошибки при `npm install`** — проверьте интернет; при блокировке npm смените регистр или используйте VPN.
3. **Порт 3000 занят** — запуск на другом порту: `npm run dev -- -p 3001`
4. Пришлите текст ошибки из терминала — по нему можно точнее подсказать.
